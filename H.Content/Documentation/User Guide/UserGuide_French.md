# Chapitre 2  Démarrage de Holos

Holos fonctionne selon l’un des deux modes appelés « De base » et « Avancé ». Comme leur nom l’indique, ils permettent d’adapter l’expérience Holos à des besoins différents. Si vous n’avez pas besoin de toutes les fonctions offertes par Holos et préférez obtenir un résumé des émissions provenant de votre ferme, il est recommandé d’utiliser le mode de base. Si vous voulez tirer le maximum de Holos et obtenir des résultats détaillés, le mode avancé est conçu pour vous.

Holos peut gérer plusieurs fermes créées par l’utilisateur, chacune représentant différents scénarios agricoles. L’utilisateur peut expérimenter ce que les pratiques de gestion font bien ensemble ou déterminer la combinaison de cultures à utiliser en fonction de leur rendement potentiel et de leurs émissions de gaz à effet de serre.

## Formatage du fichier personnalisé de données climatiques


Pour importer vos propres données climatiques dans Holos, le fichier doit être un fichier de valeurs séparées par des virgules (CSV) et avoir l’extension de fichier « .csv ». Il doit également contenir les colonnes suivantes :
* **Année** – L’année à laquelle se rapportent les données climatiques.
* **Jour julien** – La journée selon le calendrier julien.
* **Température moyenne quotidienne de l’air** – La température moyenne de la journée.
* **Précipitations quotidiennes moyennes** – Les précipitations moyennes de la journée.
* **Évapotranspiration potentielle quotidienne moyenne (TEP)** – Évapotranspiration potentielle moyenne de la journée.

#### Sauvegarder

1.	Si vous avez déjà créé une ferme, mais souhaitez l’enregistrer sous un autre nom et la modifier, vous pouvez sélectionner **Enregistrer sous**. Comme pour d’autres produits, vous pouvez créer une copie au moyen de **Enregistrer sous**. Il vous suffit de sélectionner E**nregistrer sous** à l’onglet Fichier et d’entrer un nom pour enregistrer votre nouvelle ferme.

2.	Si le nom existe déjà, Holos vous en avertira et vous demandera de le changer.
3.	Vous pouvez maintenant accéder à cette nouvelle ferme copiée de la même façon que vous ouvririez une ferme à partir de l’onglet Fichier.

### Gestion des fermes

Dans cette section, vous pouvez voir toutes les fermes qui figurent dans le fichier sur lequel vous travaillez actuellement. Vous verrez une liste de toutes les fermes avec leurs données. **Vous pouvez supprimer vos fermes dans cet onglet et modifier certaines de leurs données.**
1.	Cliquez sur l’icône **Fermes** sous l’en-tête Fichier en haut de l’écran.
2.	Une fenêtre s’ouvrira pour afficher chaque ferme dans le fichier. Il est possible de modifier chaque section de la ferme, **Nom, Commentaires, Date de création et Date de modification**, en cliquant sur la case appropriée et en modifiant son contenu.

## Modèle annuel et pluriannuel

*	Holos travaille avec une ferme selon deux modes, **Annuel** et **Pluriannuel**. Le mode de base est le mode **annuel**, certains boutons et menus étant masqués pour plus de simplicité. Holos démarre par défaut toute ferme en mode **Annuel**. Comme son nom l’indique, le mode **Annuel** ne fournit des données que pour une seule année. Le mode Pluriannuel permet de mettre en place un **système avancé de modélisation du carbone** sur une période allant de 1985 à nos jours.

*	Le mode **Annuel** est le plus simple pour utiliser Holos; il suffit d’ajouter des composantes à votre ferme et d’appuyer sur **Suivant** pour charger les résultats. Il s’agit du mode par défaut pour créer une nouvelle ferme dans Holos.

### Composante des brise-vent

-   Un **brise-vent** est constitué de rangées d'arbres dont chacune contient divers types d'arbres, comme le caragana, le frêne vert, le peuplier hybride, etc.

-   **Une rangée brise-vent peut contenir plus d'un type d'arbre et il faut remplir des champs de données pour chaque type d'arbre.**

-   Dans cette section, nous parcourrons étape par étape la façon dont nous pouvons modifier les données d'une composante de brise-vent en fonction de votre exploitation.

-   **Si vous n'avez pas créé de composante de brise-vent**, veuillez cliquer ici [Introduction à la sélection de omposantes](#présentation-de-la-sélection-des-composantes) pour savoir comment ajouter une composante à votre ferme.

#### Modification des rangées de brise-vent

-   **Chaque rangée d'un brise-vent est composée de divers types d'arbres** et **chaque type d'arbre a ses propres champs de données** qui doivent être remplis, comme le montre **la figure 6.3**. **Cette question sera traitée dans la section Modification des données sur le type d'arbre.**

-   Chaque ligne comporte un champ de données pour le **Nom de la rangée et la longueur de la rangée (m)** et se trouve au-dessus de chaque rangée, comme le **montre la figure 6.3**.

Par exemple, à la figure 6.3, la première rangée de notre brise-vent est composée d'un caragana et d'un peuplier hybride; par conséquent, le nom de la rangée sera automatiquement modifié pour devenir Caragana, Peuplier hybride.

Si vous ne connaissez pas la circonférence moyenne de votre type d'arbre, cliquez sur le bouton **Calculatrice** à côté du champ Circonférence moyenne. -   Les composantes animales sont réparties dans les catégories suivantes : **Production de bœuf, Produits laitiers, Porcs, Volaille, Moutons et Autres animaux d'élevage.**

## Survol

-   Pour afficher cet écran, assurez-vous d'**activer le mode pluriannuel** en **mode avancé** et de **désactiver les résultats simplifiés** dans la barre d'outils.

-   Cet écran présente une grille détaillée liée aux champs de votre ferme, comme l'**apport en carbone**, le **rendement** et d'**autres facteurs** pour chaque année de vos systèmes de production que vous avez créés dans le **calendrier**.

-   Nous avons d'abord vu le « **plateau de données** » (voir la figure 8.1) à gauche de la grille des systèmes de culture.
    -   Cliquez sur « **Activer les colonnes** » à gauche de la grille de données pour agrandir un plateau des colonnes de données disponibles.
    -   Cliquez de nouveau sur « Activer les colonnes » pour réduire le plateau sur le côté.
    -   Le plateau vous permet d'afficher ou de masquer différentes colonnes dans la grille. Votre choix des colonnes à masquer persistera entre les séances et les fermes (c.‑à‑d. les colonnes de données masquées dans une ferme seront également masquées dans d'autres fermes nouvelles ou existantes).
    -   Les données sont disponibles dans d'autres vues lorsqu'on utilise exactement la même fonctionnalité pour chaque cas.
    
-   Si vous êtes arrivé à cet écran et que vous êtes retourné pour modifier quelque chose à propos de vos champs, vous devrez peut-être recharger les données pour les tenir à jour en fonction de vos changements. Pour effectuer le rechargement, cliquez sur « **Recharger les données de l'écran précédent** » dans le coin supérieur gauche de la vue.

-   Vous pouvez modifier la façon dont l'écran affiche les rendements de vos champs à l'aide du menu déroulant à côté de « **Méthode d'attribution du rendement** » **(la valeur par défaut est « Rendement moyen »).**

-   Comme pour les autres grilles de Holos, vous pouvez trier et filtrer chaque colonne affichée. Tout ce qui se trouve à droite de la colonne Rendement peut être modifié à la main.

# Chapitre 10 - Interface de ligne de commande

## Qu'est-ce qu'une interface de ligne de commande?

-   L'interface de ligne de commande (CLI) Holos vous permet de traiter efficacement de grandes quantités de données correspondant aux diverses activités de votre ferme.

-   L'interface exige que l'utilisateur **crée des fichiers Excel d'entrée de données** pour plusieurs fermes et leurs composantes respectives (c.-à-d. champs, naissage, porcs, etc.).

-   Les fichiers Excel d'entrée de données doivent être enregistrés en format CSV. **Microsoft Excel est recommandé** étant donné que d'autres logiciels tiers (tels qu'OpenOffice) sont connus pour
causer des erreurs de saisie.

-   Les fichiers d'entrée ne peuvent pas être ouverts dans un autre programme pendant l'exécution de la CLI.

-   Dans cette section, nous verrons comment **configurer la CLI**, comment **modifier les fichiers de paramètres**, comment **créer des fichiers de données pour chaque composante et comment utiliser la
CLI.**

## Configuration de l'interface de ligne de commande

Dans cette section, nous passerons en revue les étapes à suivre pour bien configurer la CLI en ce qui concerne la navigation dans le répertoire Holos, la modification des fichiers de paramètres globaux, la
création ou la modification de fichiers Excel d'entrée de données et l'ajout d'un nouveau dossier à la CLI, pour chacune des composantes de votre ferme.

<br>

### Le dossier Fermes dans la CLI

-   L'interface de ligne de commande Holos exige que l'utilisateur crée un dossier « Fermes » dans lequel toutes les données liées aux fermes seront stockées.

-   Ce dossier Fermes se trouve dans un dossier général qui contient toutes les fermes individuelles avec lesquelles l'utilisateur veut travailler.

-   Chaque dossier de ferme individuelle contient des dossiers différents pour chacune des composantes de votre ferme, ainsi qu'un fichier de paramètres globaux pour cette ferme (où vous pouvez établir vos valeurs par défaut globales pour chaque ferme).

-   De plus, chaque dossier de composante contient des fichiers Excel qui stockent les données d'entrée pour chacune de vos activités (c.-à-d. des fichiers d'entrée de données pour un champ ou une composante naissage dans votre ferme).

<br>

Après avoir spécifié votre dossier Fermes général, Holos créera un dossier par défaut dans ce dossier général, qui sera appelé « **FermeExempleHolos** ».

##### Valeurs d'entrée de données spéciales

-   **Zone de rusticité**: La convention d'appellation est « H(0-9)(a-b) ». Par exemple, une zone de rusticité valide serait « H3a » ou « H3b » ou « H9a », où la première lettre serait toujours « H » et, pour le numéro ou la lettre suivant, vous pouvez choisir une fourchette entre 0 et 9 et a ou b pour représenter la zone de rusticité appropriée pour votre ferme.

-   **Espèces**: Caragana, frêne vert, peuplier hybride, érable du Manitoba, pin sylvestre, épinette blanche, feuillu moyen, conifère moyen.

##### Cultures vivaces

-   Si la culture est **vivace, il y a trois champs de données importants qui doivent être remplis correctement** : année de la récolte, identifiant du peuplement vivace et durée du peuplement vivace (années).
    
-   L'identifiant du peuplement vivace prend la forme 00000000-0000-0000-0000-000000000000, où les zéros peuvent être le caractère que vous voulez. Un identifiant de peuplement simple peut être 00000000-0000-0000-0000-000000000001.
    
-   **Pour créer une durée de peuplement vivace**, veuillez créer la ou les rangées appropriées pour chaque année, mais **changez l'année de récolte et l'année de peuplement vivace en conséquence et reportez-vous à la culture vivace appropriée au moyen de l'identifiant de peuplement vivace**. (p. ex., si nous faisons référence à Culturevivace1, et qu'elle a une durée de peuplement vivace de quatre ans, nous aurons quatre lignes Excel qui se rapportent à l'identifiant de peuplement vivace de 00000000-0000-0000-0000-000000000001, mais avec des valeurs différentes pour l'année-récolte (2017, 2018, 2019 et 2020) et l'année de peuplement vivace (1, 2, 3, 4) et des valeurs de données variables pour les données sur les cultures.
    
-   Cela nous permet de regrouper avec précision les données sur les **cultures vivaces** pour chaque année.

-   **Pour chaque type de culture vivace, veuillez attribuer un identifiant de peuplement vivace unique**. P. ex., si le peuplement vivace était un mélange de foin de brome inerme et de luzerne; le brome inerme aurait un identifiant de peuplement vivace unique et la luzerne aurait un identifiant de peuplement vivace unique.

-   Par conséquent, pour chacune des cultures vivaces de votre champ, veuillez suivre la même procédure que celle indiquée ci-dessus pour remplir le reste du fichier d'entrée de données Excel.

#### Composantes animales

-   Les composantes animales comprennent les groupes du **porc, du boeuf, des bovins laitiers, du mouton, de la volaille et d'autres animaux d'élevage**.

-   Les composantes animales **comprennent toutes différents groupes d'animaux**

-   Chaque groupe d'animaux a sa ou ses propres périodes de gestion qui consistent en des données sur les **régimes alimentaires, les bâtiments d'élevage et le fumier** **pour une période de gestion.**

-   **Vous pouvez avoir plusieurs périodes de gestion pour un groupe d'animaux.**

-   **Si vous avez plusieurs périodes de gestion pour un groupe d'animaux**, veuillez ajouter une autre rangée faisant référence au même groupe d'animaux (**même nom de groupe et même type de groupe**), mais modifiez les données de la période de gestion en conséquence.

-   **Vous ne pouvez pas avoir deux types d'animaux différents qui appartiennent au même groupe**. Par exemple, si vous examinez une composante porcine et que vous avez un groupe appelé « GroupeDémarragePorcs1 ». Ce groupe ne peut pas faire référence à plus d'un type d'animal (c.-à-d. seulement Démarrage des porcs, Finition ou Croissance des porcs) -- vous pouvez appeler le groupe
comme vous le voulez, **un groupe d'animaux ne peut pas faire référence à plus d'un type d'animal.**

-   Certaines **composantes animales** auront des données supplémentaires qui sont propres à leur type d'animal, comme la **production laitière** pour les animaux laitiers ou la **production de laine** pour les moutons.

-   Dans cette section, nous passerons en revue les différents types de composantes animales et leurs fichiers de données.

-   Vous trouverez **ci-dessous** un exemple des catégories que vous verrez dans un fichier de données sur les composantes animales.

-   Les en-têtes **Type de bâtiment d'élevage** et **Gestion du fumier** ne sont présentés ci-dessous qu'à titre d'exemple pour faciliter la catégorisation. Elles ne seront **PAS** présentes dans vos fichiers d'entrée de données.

## Utilisation de l'interface de ligne de commande

-   Si vous n'avez pas configuré vos dossiers et fichiers d'entrée de données pour les composantes de votre ferme, veuillez cliquer ici : [Configuration de l'interface de ligne de commande](#configuration-de-linterface-de-ligne-de-commande)

-   Si vous ne savez pas comment modifier votre fichier de paramètres pour votre ferme, veuillez cliquer ici : [Modification des fichiers de paramètres d'une ferme](#modification-des-fichiers-de-paramètres)

-   Veuillez consulter la liste de vérification de la CLI ici : [Liste de vérification de l'interface de ligne de commande Holos](#liste-de-vérification-de-linterface-de-ligne-de-commande)
-   Autrement, les fichiers correspondant à votre ferme sont prêts à être traités et cette section vous indiquera comment exécuter l'interface de ligne de commande.

1.  Dans votre fichier de paramètres pour chaque ferme, ajustez les paramètres en fonction de vos activités. Si vous ne savez pas ce qu'est un fichier de paramètres, cliquez ici : [Modification des
fichiers de paramètres](#modification-des-fichiers-de-paramètres).

2.  La première fois que vous démarrerez l'interface de ligne de commande, vous serez invité à entrer l'emplacement de votre dossier « Fermes » (il peut s'agir de n'importe quel dossier que vous créez sur votre ordinateur).

    -   Les autres fois que vous démarrez l'interface de ligne de commande Holos, vous devez indiquer si vous souhaitez utiliser votre emplacement précédemment entré ou entrer un nouvel emplacement.
    
    - Reportez-vous à la section [Créer un nouveau dossier Fermes](#créer-un-nouveau-dossier-fermes) pour obtenir de l'aide sur la création d'un dossier de ferme initiales et pour en savoir plus sur les invites subséquentes.

3.  Vous serez invité à sélectionner une unité de mesure pour le programme à exécuter (Métrique = 1, Impérial = 2). Veuillez faire un choix

4.  L'interface de ligne de commande traitera maintenant chacune des fermes que vous avez créées et les fichiers d'entrée de données de leurs composantes correspondantes.

-   Une fois le traitement et l'analyse terminés, vous verrez un message « Conversion terminée » dans la CLI.

5.  Les résultats pour votre ferme seront exportés dans le répertoire de base de votre répertoire « Fermes » dans un dossier « Extrants » à l'étape 1.

## Résultats de l'interface de ligne de commande

-   L'interface de ligne de commande produira les **résultats pour chaque ferme** que vous avez créée et ses composantes respectives, ainsi que les **résultats totaux pour toutes les fermes combinées.**

-   Les résultats seront affichés dans un fichier Excel et correspondront aux catégories suivantes : **Émissions d'équivalent du dioxyde de carbone, Émissions de gaz à effet de serre, Estimations de la production et Estimations des aliments.**

-   Les **émissions d'équivalent du dioxyde de carbone** sont exprimées en **mégagrammes (Mg)** et les **émissions de gaz à effet de serre**, en **kilogrammes (kg).**

-   **Si vous avez plusieurs fichiers de paramètres pour une ferme**, il y aura des fichiers de résultats pour chaque fichier de paramètres dans cette ferme.

-   Vous trouverez ces résultats dans le dossier « **Extrants** » du répertoire de base de l'interface de ligne de commande Holos.

-   **Si vous cherchez des résultats plus précis pour chaque ferme, veuillez vous rendre au dossier d'extrants approprié.**

### Total des résultats (par année)

-   Les **résultats totaux (toutes les fermes)** calculeront les estimations appropriées pour chaque ferme et leurs composantes et groupes d'animaux respectifs et les afficheront dans un fichier Excel. Vous pourrez voir les totaux pour chaque ferme et les totaux pour toutes les fermes combinées **chaque année.**

-   **Chaque composante** d'une ferme **est catégorisée** comme suit : **Production de bœuf, bovins laitiers, autres animaux d'élevage, volaille, moutons et porcs.**

-   **Si les données ne s'appliquent pas**, le champ de données affichera « S. O. ».

-   **Chaque catégorie de composantes** comprend des **composantes**, chacune étant constituée de **groupes d'animaux.**

-   Les résultats pour chacune de ces catégories de composantes et leurs composantes respectives seront calculés et affichés pour chaque ferme.

-   Ces résultats se trouveront dans le dossier « **Extrants** » situé dans le répertoire de base de votre dossier « **Fermes** » dans le dossier « **RésultatsTotauxPourToutesLesFermes** ».

### Émissions d'équivalent en dioxyde de carbone

-   Holos calculera les émissions d'équivalent en dioxyde de carbone pour chaque ferme et leurs composantes respectives.

-   Les résultats comprendront les valeurs d'émission calculées suivantes : **Émissions de CH~4~ entérique, CH~4~ du fumier, oxyde nitreux (directes), oxyde nitreux (indirectes), dioxyde de carbone par énergie et dioxyde de carbone.**

-   Le **dioxyde de carbone provenant des énergies** est calculé pour **chaque groupe d'animaux** (c.-à-d. les bouvillons de finition). **Par conséquent**, vous verrez « **S.O.** » pour les champs de données qui ne s'appliquent pas.

### Émission de gaz à effet de serre

-   Holos calculera les émissions de gaz à effet de serre pour chaque ferme et leurs composantes respectives.

-   Les résultats comprendront les valeurs d'émission calculées suivantes : **Émissions de CH~4~ entérique, CH~4~ du fumier, oxyde nitreux (directes), oxyde nitreux (indirectes), dioxyde de carbone par énergie et dioxyde de carbone.**

-   Le **dioxyde de carbone provenant des énergies** est calculé pour **chaque groupe d'animaux** (c.-à-d. les bouvillons de finition). **Par conséquent**, vous verrez « S.O. » pour les champs de données qui ne s'appliquent pas.

-   **Toutefois, ces valeurs ne sont pas converties en valeurs d'équivalent du dioxyde de carbone.**

### Estimations de la production

-   Holos calculera les estimations de la production pour chaque ferme et leurs composantes respectives.

-   Les résultats comprendront les estimations calculées suivantes des valeurs de production : **Récolte, superficie, fumier disponible pour l'épandage sur le sol, bœuf produit, agneau produit, lait produit et production de lait normalisé en m.g. et protéines.**

### Estimations des aliments pour animaux

-   Holos calculera les estimations de la production pour chaque ferme et leurs composantes respectives.

-   Les résultats comprendront la valeur estimative calculée suivante pour les aliments : **Ingestion de matières sèches.**

-   Pour chaque composante de la gestion des terres, il y aura un fichier de résultats spécifique pour chaque fichier de données de composante de la gestion des terres que vous aurez créé. **Ces résultats sont propres à la composante qui vous intéresse** et nous allons les passer en revue dans cette section.

#### Résultats de la composante Brise-vent

#### Résultats de la composante Champ

## Liste de vérification de l'interface de ligne de commande

-   L'interface de ligne de commande exige que les données d'entrée soient correctement formatées et que les composantes appropriées soient créées pour que les données puissent être traitées.

-   Cette section vous permet de vous assurer que vous ne rencontrez aucune erreur lors de la configuration de l'interface de ligne de commande.

-   Si vous rencontrez des erreurs, veuillez cliquer ici : [Guide d'erreur pour l'interface de ligne de commande](#guide-derreur-pour-linterface-de-ligne-de-commande)

<br>

### Fichiers de données correctement remplis

-   Remplir conformément aux directives de la section : [Comment modifier les fichiers Excel d'entrée de données des composantes](#modification-des-fichiers-excel-dentrée-de-données-des-composantes).
-   **N'oubliez pas que chaque fichier représente l'une des composantes de votre ferme**. Par exemple, si vous avez deux brise-vent que vous aimeriez représenter comme fichiers Excel d'entrée de données, vous devrez créer deux fichiers Excel d'entrée de données, un pour chaque brise-vent et vous assurer que leurs noms sont uniques.
    
-   **Ne modifiez pas ou ne supprimez pas les en-têtes** des fichiers Excel d'entrée de données, sinon l'interface de ligne de commande cessera le traitement.
    
-   **Entrez les données correspondant aux lignes directrices pour chaque composante**; il y a des **en-têtes spéciaux qui nécessitent des données précises** que vous pouvez choisir dans chaque ligne directrice. Assurez-vous d'orthographier vos données d'entrée correctement!
    
-   **Si vous rencontrez l'une de ces erreurs**, veuillez accéder au guide des composantes approprié sous : [Modification des fichiers d'entrée de données des composantes](#modification-des-fichiers-excel-dentrée-de-données-des-composantes). Chacune des sections contiendra un guide pour les champs de données spéciaux, indiquant quelles données sont valides pour ce champ.
    
-   **N'entrez pas de valeurs négatives lorsqu'elles ne sont pas logiques**. Par exemple, si vous réglez l'« année d'observation » à « -1996 », l'interface de ligne de commande cessera le traitement.
    
-   **Ne laissez pas de champs de données vides,** : l'interface de ligne de commande cessera le traitement s'il n'y a pas de données.
    
-   **Pour certains champs de données, si l'en-tête ne s'applique pas, réglez l'entrée de données à « S.O. »**, par exemple dans une composante champ, si vous avez une récolte annuelle et que vous tombez sur les en-têtes « **Identifiant de peuplement vivace** », « **Durée de peuplement vivace** » et « **Année en peuplement vivace** », veuillez régler ces champs à « **S.O.** ».

<br>

### Noms uniques des composantes

Par exemple, dans le répertoire des brise-vent, si vous avez deux fichiers Excel : Brise-vent1.csv et Brise-vent2.csv et qu'ils contiennent tous deux des données pour la composante « **Brise-vent Exemple1** », l'interface de ligne de commande Holos ne traitera pas les données parce que leurs noms ne sont pas uniques.

<br>

### Références valides à d'autres composantes

L'utilisateur doit s'assurer que les références aux autres composantes sont valides et que la composante est en fait un fichier de données.

-   Assurez-vous que si l'une de vos composantes nécessite une référence à une autre composante que vous avez établie, que la composante en question a le bon fichier Excel d'entrée de données créé dans le bon dossier de ferme et de composante et que vous mentionnez le nom de la composante (**PAS LE NOM DU FICHIER**). si vous devez vous y référer dans l'un de vos champs de saisie de données pour une autre composante.

-   **Par exemple**, dans une composante **porcine**, vous avez choisi comme **type de bâtiment d'élevage** « pâturage » et le pâturage auquel vous aimeriez vous référer est un champ personnalisé qui reflète votre exploitation agricole. **Veuillez vous assurer de créer un fichier Excel d'entrée de données avec les données qui reflètent votre champ personnalisé dans le dossier des champs de la ferme choisi et de** vous assurer que le champ de données du nom de la composante pour chaque fichier d'entrée de données est unique (**vous ne pouvez pas avoir de composantes portant le même nom**).

-   Par conséquent, si le **nom du champ** personnalisé que vous avez créé est « **NomChampExemple1** », chaque fois que vous souhaitez vous référer à ce champ spécifique dans un composante qui peut avoir une référence à un champ, vous devez définir le champ de données comme « NomChampExemple1 ».

-   Par conséquent, pour le champ de données Pâturage de la composante Porcs, vous établiriez le champ de saisie de données à « NomChampExemple1 » si vous souhaitez vous référer à ce champ personnalisé.

<br>

### Les noms des répertoires de composantes générés par la CLI ne sont pas modifiés

L'utilisateur doit s'assurer de ne pas modifier les noms des répertoires de composantes dans chaque ferme.

Si les noms des composantes ne sont pas valides, les données ne seront pas traitées. Un message s'affiche pour indiquer quel répertoire de composantes n'est pas valide.

<br>

# Chapitre 11 Questions courantes


## Comment puis-je activer le mode avance

Si vous avez démarré la ferme en mode de base et que vous souhaitez passer au mode avancé, le processus est extrêmement simple.

1.	Cliquez sur Paramètres dans la barre de menu supérieure.
2.	Sélectionnez « **Valeurs par défaut de la ferme** ».
3.	Sélectionnez « **Mode avancé** » sous l’onglet « Ferme ».

## Comment puis-je activer le mode pluriannuel

Le mode pluriannuel n’est disponible que si vous êtes actuellement en mode avancé. Si vous êtes actuellement en mode de base, passez au mode avancé. Ensuite, cherchez un bouton à bascule dans le coin supérieur droit du programme Holos. Vous pouvez utiliser ce bouton pour passer d’un mode à l’autre.

## Comment basculer entre les modèles de carbone de niveau 2 du GIEC et MIBC

Holos utilise par défaut le modèle de carbone de niveau 2 du GIEC, mais il permet à l’utilisateur de passer au modèle de carbone MIBC. Pour passer d’un modèle à l’autre :
1.	Cliquez sur Paramètres dans la barre de menu supérieure.
2.	Sélectionnez « **Paramètres de l’utilisateur** ».
3.	Dans la liste déroulante « **Stratégie de modélisation du carbone** », sélectionnez le modèle de carbone que vous voulez utiliser dans vos calculs.

## Comment puis-je ajouter une nouvelle ferme à mon dossier CLI actuel

### Méthode 1 : Copier un dossier de ferme vide existant.

1.	Dans votre dossier « Fermes » actuel, créez un nouveau dossier pour votre nouvelle ferme avec un nom approprié.
2.	Copiez tous les fichiers d’une ferme existante et collez-les dans le nouveau dossier que vous venez de créer.
3.	Modifiez les fichiers d'entrée de données au besoin.

### Méthode 2 : Exécuter à nouveau la CLI.

1.	Démarrer la CLI Holos.
2.	Vous serez invité à utiliser le dossier Fermes précédemment utilisé. Sélectionnez oui.
3.	Sélectionnez les unités de mesure appropriées dans l’invite suivante.
